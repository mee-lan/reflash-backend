// CHANGES MADE:

i. FlashcardRepository.java
   - Removed the old three generic methods (getNewCards, getLearningRelearningCards, getReviewCards) which had
     parameter name bugs (@Param("id") vs :id mismatch) and no access control.
   - Added six new role-scoped query methods (two sets of three):
       * getNewCardsForStudent / getNewCardsForTeacher
       * getLearningRelearningCardsForStudent / getLearningRelearningCardsForTeacher
       * getReviewCardsForStudent / getReviewCardsForTeacher
   - Each student method JOINs course_student and checks s.id = :userId.
   - Each teacher method JOINs course_teacher and checks t.id = :userId.
   - Decision: Two separate endpoints (student/teacher) chosen over a single endpoint because
     the security models differ (ROLE_STUDENT vs ROLE_TEACHER) and two focused sets of queries
     are cleaner than dynamic branching in a single JPQL query.

ii. FlashcardService.java
   - getDeck() now accepts (Integer deckId, Integer userId, String role).
   - Uses DeckRepository.findById(deckId) to fetch the deck and read deck.crt.
   - Computes 'today' as the number of full days elapsed since the deck's creation epoch
     (midnight UTC): today = (todayMidnightEpochSeconds - deck.crt) / 86400.
   - currentTime for learning/relearning cards uses System.currentTimeMillis() / 1000
     (epoch seconds, matching how Flashcard.crt is stored).
   - Branches on role to call the student or teacher variants of each repository query.
   - Combines all three card lists, maps each Flashcard -> FlashcardDto, wraps in DeckDto.

iii. FlashcardDto.java
   - Fully implemented following the CourseStudentDto pattern.
   - Fields: id, note (Note entity), crt, type (CardType), queue (CardQueue),
             ivl, factor, reps, lapses, left, due.
   - Constructor: FlashcardDto(Flashcard flashcard).

iv. DeckDto.java
   - Fully implemented.
   - Fields: id, courseName (from deck.course.getCourseName()), deckName, flashcards (List<FlashcardDto>).
   - Constructor: DeckDto(Deck deck, List<FlashcardDto> flashcards).

v. FlashcardController.java
   - Replaced the old stub with two working endpoints:
       GET /api/student/flashcards  — @PreAuthorize("hasRole('STUDENT')")
                                       @RequestParam Integer deckId
                                       calls getDeck(deckId, student.getId(), "STUDENT")
       GET /api/teacher/flashcards  — @PreAuthorize("hasRole('TEACHER')")
                                       @RequestParam Integer deckId
                                       calls getDeck(deckId, teacher.getId(), "TEACHER")
   - Both return ResponseEntity<ApiResponse> wrapping the DeckDto.
